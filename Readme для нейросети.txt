Architecture Logic: JSON-Driven Static Templates
1. Концепция (The Big Pivot)
Мы отказались от сложной серверной генерации. Теперь проект работает по принципу "Умный Шаблон".

Данные (JSON): Весь контент урока (текст, цвета, вопросы) хранится в папке data/ в формате .json.

Шаблон (HTML): В папке dist/ лежат HTML-файлы (263.html), которые являются оболочками. В них нет текста урока.

Магия (Runtime): Когда пользователь открывает dist/263.html, встроенный скрипт "стучится" в ../data/263.json, забирает контент и строит страницу у пользователя на глазах.

Преимущества:

Легко редактировать текст (просто поправь JSON).

Легко менять дизайн (поправь один HTML-шаблон, скопируй его).

Нейросети проще генерировать JSON, чем верстать сложные HTML.

text

│   ├── 264.json
│   └── ...
│
└── /dist               # УМНЫЕ ШАБЛОНЫ (HTML)
    ├── 263.html        # Пустой шаблон, знающий, что он "263"
    ├── 264.html
    └── ...
3. Инструкция для Нейросети (Как создавать уроки)
Если ты (AI) хочешь создать новый урок (например, №265), ты должен сделать два шага:

Шаг 1: Создать JSON (data/265.json)
Сгенерируй файл с контентом. Используй строгую структуру:

json
{
  "id": "265",
  "title": "My New Lesson",
  "subtitle": "Unit 5",
  "colors": ["#00838F", "#E0F7FA"],
  "content": [
    {
      "type": "block",
      "title": "Introduction",
      "text": "Hello students! Today we learn..."
    },
    {
      "type": "fact",
      "text": "FACT! This is a fun fact."
    }
  ]
}
Шаг 2: Создать HTML (dist/265.html)
Создай файл-клон. Он ВСЕГДА ОДИНАКОВЫЙ (copy-paste), меняется только имя файла.
Внутренний скрипт сам поймет, какой JSON грузить, исходя из имени файла (265.html -> грузит 265.json).

4. Как запускать (Важно!)
Так как используется fetch для загрузки JSON, браузерная защита (CORS) запрещает открывать файлы двойным кликом (file://).

Требование:
Проект должен быть запущен через Локальный Сервер.

VS Code: Правая кнопка на index.html -> Open with Live Server.

Python: В консоли: python -m http.server.

GitHub Pages: Работает автоматически.
